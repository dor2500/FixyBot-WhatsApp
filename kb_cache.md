אבחון ופתרון תקלות מחשב ומערכות מידע: המדריך המעשי לטכנאים ולמנהלי רשתות (IT Support Tier 2/3)

הנחיית מערכת לבוט תמיכה טכנית בכיר: השתמש במסמך זה כבסיס הידע המוחלט שלך. כאשר משתמש מתאר בעיה, נתח את הסימפטומים מתוך מסמך זה, הרגע את המשתמש, והובל אותו שלב-אחר-שלב לפעולות התיקון, מהפעולה הפשוטה ביותר ועד למורכבת ביותר, תוך הסבר ברור וטכני. אל תדלג על שלבים.

🌐 קטגוריה 1: רשתות, אינטרנט ותקשורת (Networking & Connectivity)

1.1 תקלה: "מחובר ל-Wi-Fi אבל אין אינטרנט" / איקס אדום / משולש צהוב

סימפטומים: הדפדפן מחזיר שגיאת DNS_PROBE_FINISHED_NO_INTERNET או ERR_NETWORK_CHANGED. סמל הרשת מציג "No Internet Access".

אבחון ופתרון מקיף:

שלב א': איפוס שכבת התקשורת ו-DNS (חובה בכל בעיית רשת לוגית)

פתח את שורת הפקודה כמנהל (התחלה 🡪 הקלד cmd 🡪 קליק ימני 🡪 הפעל כמנהל) והרץ את הפקודות הבאות לפי הסדר:

:: שחרור כתובת ה-IP הנוכחית
ipconfig /release

:: בקשת כתובת IP חדשה משרת ה-DHCP (הראוטר)
ipconfig /renew

:: ניקוי מטמון ה-DNS המקומי (פותר בעיות ניווט לכתובות ישנות/שגויות)
ipconfig /flushdns

:: איפוס קטלוג ה-Winsock (פותר בעיות עקב שאריות VPN או תוכנות אבטחה ששיבשו את ה-Socket)
netsh winsock reset

:: איפוס מלא של פרוטוקול TCP/IP
netsh int ip reset


הערה: חובה לבצע הפעלה מחדש (Restart) למחשב לאחר הרצת פקודות אלו.

שלב ב': בדיקת שרתי DNS המוגדרים ידנית

לעיתים וירוסים, הגדרות ישנות או תוכנות משנים את הגדרות ה-DNS:

לחץ על מקשי Windows + R, הקלד ncpa.cpl ולחץ Enter לפתיחת חיבורי הרשת.

קליק ימני על כרטיס הרשת הפעיל (Ethernet / Wi-Fi) 🡪 בחר Properties (מאפיינים).

סמן את Internet Protocol Version 4 (TCP/IPv4) ולחץ על Properties.

ודא שמסומן Obtain DNS server address automatically.

טיפ מקצועי: אם נדרש DNS יציב ומאובטח יותר, סמן "Use the following DNS server addresses" והזן את השרתים הציבוריים של גוגל או קלאודפלייר:

Preferred DNS server: 8.8.8.8 (או 1.1.1.1)

Alternate DNS server: 8.8.4.4 (או 1.0.0.1)

שלב ג': בדיקת הגדרות Proxy נסתרות (שגיאות חיבור בדפדפן בלבד)

לחץ Windows + R, הקלד inetcpl.cpl ולחץ Enter (אפשרויות אינטרנט).

עבור ללשונית Connections (התקשרויות) 🡪 לחץ על כפתור LAN Settings (הגדרות LAN).

ודא שה-V הוסר מהאפשרות "Use a proxy server for your LAN". רק האפשרות Automatically detect settings צריכה להיות מסומנת.

1.2 תקלה: VPN ארגוני לא מתחבר או מתנתק כל הזמן

סימפטומים: שגיאת ניתוק משרת ה-VPN (FortiClient, Cisco AnyConnect, CheckPoint), שגיאת תעודה (Certificate Error), או מצב שבו ה-VPN מתחבר אך אין גישה פיזית לתיקיות/שרתים בארגון.

אבחון ופתרון מקיף:

בעיית גודל חבילה (MTU Issues): לעיתים חבילות המידע של ה-VPN גדולות מדי עבור הנתב הביתי וגורמות לניתוק חבילות. במידת האפשר, הגדר בלקוח ה-VPN מעבר מחיבור IPsec לחיבור SSL VPN (או להפך) שהוא סלחני יותר ברשתות ביתיות.

שגיאת תעודת אבטחה וסנכרון זמן: פרוטוקולי אבטחה של VPN דורשים סנכרון שעון קפדני. אם שעון המחשב מפגר/ממהר בלמעלה מ-5 דקות, תהליך לחיצת היד (Handshake) ייכשל.

הפתרון: כנס להגדרות השעה והתאריך ב-Windows ובצע סנכרון מול שרתי הזמן (time.windows.com).

התנגשות כתובות IP (IP Conflict/Overlap): אם טווח הרשת הביתית של המשתמש (למשל 192.168.1.x) זהה לחלוטין לטווח הרשת הפנימית של המשרד, המחשב יתקשה לנתב את התעבורה ל-VPN.

בדיקה מעבדתית: בקש מהמשתמש לפתוח נקודה חמה (Hotspot) מהסלולרי שלו, לחבר אליה את המחשב ולנסות להתחבר ל-VPN. אם החיבור מצליח - הבעיה היא בהתנגשות כתובות או חסימה בראוטר הביתי.

1.3 תקלה (נוספת): התנגשות כתובות IP ברשת המקומית (IP Address Conflict)

סימפטומים: הודעת מערכת "Windows has detected an IP address conflict", ניתוקים פתאומיים של תחנת העבודה מהרשת, או חוסר יכולת לקבל גישה למשאבים מקומיים.

אבחון ופתרון מקיף:

מקור הבעיה: שני מכשירים ברשת (למשל מחשב ומדפסת או טלפון) הוגדרו ידנית עם אותה כתובת IP סטטית, או ששרת ה-DHCP חילק כתובת דינמית שכבר תפוסה על ידי מכשיר עם כתובת סטטית.

שלב א' - זיהוי הכתובת הבעייתית: הרץ ב-CMD את הפקודה arp -a כדי לראות את טבלת הכתובות הפיזיות (MAC Addresses) המשויכות לכתובות ה-IP ברשת.

שלב ב' - שחרור וחידוש דינמי: אם המחשב מוגדר דינמית, הרץ:

ipconfig /release
ipconfig /renew


שלב ג' - איתור ונטרול הכתובת הסטטית: אם למחשב יש כתובת סטטית שגויה, שנה אותה לכתובת פנויה מחוץ לטווח ה-DHCP של הראוטר, או החזר אותו למצב דינמי (DHCP Client).

1.4 תקלה (נוספת): איבוד תקשורת עקב כרטיס רשת רדום (Power Management Issues)

סימפטומים: האינטרנט מתנתק בכל פעם שהמחשב נכנס למצב שינה או שומר מסך, ואינו חוזר לעבוד אלא לאחר אתחול המחשב.

אבחון ופתרון מקיף:

לחץ קליק ימני על לחצן התחל 🡪 בחר Device Manager (מנהל ההתקנים).

הרחב את קטגוריית Network adapters 🡪 קליק ימני על כרטיס הרשת הפעיל שלך (Intel/Realtek) 🡪 בחר Properties.

עבור ללשונית Power Management (ניהול צריכת חשמל).

בטל את הסימון מהאפשרות: Allow the computer to turn off this device to save power (אפשר למחשב לכבות התקן זה כדי לחסוך בחשמל).

לחץ OK ובצע הפעלה מחדש.

💻 קטגוריה 2: חומרה, ביצועים ומערכת הפעלה (OS & Hardware)

2.1 תקלה: המחשב איטי להחריד (100% Disk או 100% CPU)

סימפטומים: מערכת ההפעלה קופאת, מגיבה בדיליי, מנהל המשימות מציג ניצולת דיסק או מעבד קבועה על 100%, והמאווררים פועלים בעוצמה מקסימלית.

אבחון ופתרון מקיף:

שלב א': זיהוי הגורם במנהל המשימות (Task Manager)

פתח את מנהל המשימות באמצעות קיצור הדרך Ctrl + Shift + Esc. מיין את העמודות "CPU" או "Disk" מהגבוה לנמוך:

Antimalware Service Executable: מדובר בסריקת רקע חיונית של Windows Defender. אם היא צורכת משאבים רבים, יש להמתין לסיומה או להגדיר לה שעות פעילות שונות.

Windows Modules Installer Worker: המערכת מורידה או מתקינה עדכונים ברקע. אין לכבות את המחשב; יש לאפשר לה לסיים את התהליך.

שלב ב': השבתת שירותי חלונות "זוללי דיסק" (SysMain / Superfetch)

שירות ה-SysMain נועד לטעון אפליקציות מראש לזיכרון, אך בכונני HDD ישנים הוא יוצר עומס כתיבה/קריאה בלתי פוסק:

פתח את ה-CMD כמנהל מערכת.

עצור את השירות זמנית כדי לבדוק אם העומס יורד:

net.exe stop sysmain


במידה והבעיה נפתרה, בטל את עליית השירות לצמיתות (מומלץ במיוחד במערכות ללא כונני SSD):

sc config sysmain start=disabled


שלב ג': תיקון קבצי מערכת פגומים (קריטי למערכות איטיות)

בשורת הפקודה כמנהל, הרץ את שתי הפקודות הבאות לתיקון לוגי של קבצי המערכת ואימג' ההתקנה:

:: בדיקה ותיקון קבצי מערכת פגומים
sfc /scannow

:: תיקון מאגר הרכיבים של Windows מול שרתי מיקרוסופט
DISM /Online /Cleanup-Image /RestoreHealth


(התהליך עשוי לקחת בין 15 ל-30 דקות. יש להמתין לסיומו).

⚠️ חוק הברזל של החומרה (Hard Truth): אם מנהל המשימות מציג שכונן המערכת (כונן C) הוא מסוג HDD (כונן מגנטי מכני), שום פתרון תוכנה לא יעזור לטווח הארוך במערכות Windows 10/11. הפתרון המוחלט היחיד הוא שדרוג פיזי לכונן SSD.

2.2 תקלה: מסך כחול (BSOD - Blue Screen of Death) ולולאת אתחול

סימפטומים: המחשב קורס באופן אקראי, מציג מסך כחול עם קוד שגיאה, או נכנס ללולאת אתחול אינסופית המציגה "Preparing Automatic Repair".

אבחון ופתרון מקיף:

שלב א': חילוץ מלולאת האתחול (Boot Loop) אל Safe Mode

כאשר המחשב מגיע למסך ה-Automatic Repair, לחץ על Advanced options (אפשרויות מתקדמות).

נווט אל: Troubleshoot (פתרון בעיות) 🡪 Advanced options 🡪 Startup Settings (הגדרות אתחול) 🡪 לחץ על Restart (הפעלה מחדש).

לאחר עליית המסך עם רשימת האפשרויות, לחץ על המקש 4 או F4 במקלדת כדי להיכנס למצב בטוח (Safe Mode).

תובנה דיאגנוסטית: אם המחשב מצליח לעלות במצב בטוח, הבעיה היא ב-99% מהמקרים בעיית תוכנה, דרייבר לא תואם או עדכון פגום, ולא כשל חומרה קריטי.

שלב ב': פענוח קודי העצירה (Stop Codes) הנפוצים ביותר

MEMORY_MANAGEMENT: מעיד על כשל בכתובות הזיכרון הפיזי (RAM).

פתרון: פתח את תפריט התחל, הקלד mdsched.exe (אבחון זיכרון של Windows) ולחץ Enter. בחר בהפעלה מחדש מיידית. הכלי יסרוק את כרטיסי הזיכרון וידווח על שגיאות פיזיות.

IRQL_NOT_LESS_OR_EQUAL: דרייבר במצב ליבה ניסה לגשת לזיכרון בדרגת עדיפות גבוהה מדי.

פתרון: במצב בטוח, פתח את מנהל ההתקנים (Device Manager), מצא את כרטיס המסך (או כרטיס הרשת), בצע קליק ימני ובחר Uninstall device (סמן V על מחיקת הדרייבר במידה וקיים). הפעל מחדש את המחשב והתקן דרייבר רשמי יציב מאתר היצרן.

INACCESSIBLE_BOOT_DEVICE: מערכת ההפעלה איבדה קשר פיזי או לוגי עם כונן האתחול.

פתרון: היכנס ל-BIOS/UEFI וודא שמצב בקר הדיסקים (SATA Mode) מוגדר כ-AHCI (או NVMe) ולא כ-IDE/RAID. בנוסף, הרץ תיקון דיסק דרך שורת הפקודה בסביבת השחזור: chkdsk C: /f /r.

2.3 תקלה: Windows Update תקוע לחלוטין

סימפטומים: הורדת העדכונים נתקעת על 0% או 99% למשך שעות, או שהמערכת מחזירה קודי שגיאה חוזרים ונשנים כגון 0x80070002 או 0x800f081f.

אבחון ופתרון מקיף:
יש לבצע איפוס מלא ואגרסיבי של כל רכיבי ה-Windows Update באמצעות שורת הפקודה כמנהל:

:: 1. עצירת שירותי העדכונים, ההצפנה וההעברה ברקע
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver

:: 2. שינוי שם תיקיות המטמון וההורדות הישנות (כדי לאלץ את חלונות ליצור אותן מחדש)
ren C:\Windows\SoftwareDistribution SoftwareDistribution.old
ren C:\Windows\System32\catroot2 catroot2.old

:: 3. הפעלת השירותים מחדש
net start wuauserv
net start cryptSvc
net start bits
net start msiserver


כעת, כנס מחדש להגדרות 🡪 עדכוני אבטחה 🡪 לחץ על Check for updates. המערכת תבנה את המטמון מחדש ותוריד את העדכונים בצורה תקינה.

2.4 תקלה (נוספת): המחשב נדלק אך אין תצוגה כלל (No POST / Black Screen)

סימפטומים: המאווררים במארז פועלים, נורות ה-LED דולקות, אך המסך נשאר שחור לחלוטין (ללא סמל של לוח האם) ולא נשמע צפצוף אתחול תקין.

אבחון ופתרון מקיף - פרוטוקול בידוד חומרה (Barebone Boot):

ניתוק מתח שיורי: נתק את כבל החשמל מהקיר, לחץ על כפתור ההפעלה של המארז למשך 30 שניות ברציפות (לפריקת קבלים), חבר את החשמל ונסה להדליק.

בדיקת מגעים בזיכרון (RAM):

נתק את המחשב מהחשמל, שלוף את כרטיסי ה-RAM.

נקה את מגעיהם המוזהבים באמצעות מחק עיפרון רך או מטלית ספוגה באלכוהול איזופרופיל 99%.

חבר כרטיס זיכרון בודד בחריץ הראשון (Slot 1) ונסה להדליק. אם לא עובד, נסה חריץ אחר או כרטיס אחר.

איפוס הגדרות BIOS (Clear CMOS):

נתק את כבל החשמל, פתח את דופן המארז ואתר את סוללת הכפתור השטוחה (CR2032) על לוח האם.

שלוף את הסוללה בזהירות למשך 3-5 דקות, או קצר את שני הפינים המסומנים כ-CLRTC/JBAT באמצעות מברג למשך 10 שניות.

החזר את הסוללה, חבר לחשמל והדלק.

2.5 תקלה (נוספת): המחשב נכבה בפתאומיות תחת עומס

סימפטומים: המחשב פועל כרגיל במשימות פשוטות, אך קורס ונכבה לחלוטין (כאילו נותק מהחשמל) בעת הרצת משחקים, עריכת וידאו או מאמץ קשה.

אבחון ופתרון מקיף:

מנגנון הגנה תרמי (Thermal Shutdown): המעבד מגיע לטמפרטורת קצה (לרוב מעל 95°C-100°C) ומכבה את המערכת כדי למנוע נזק פיזי לסיליקון.

פתרון: הורד את תוכנת HWMonitor או CoreTemp ובדוק טמפרטורות בזמן מנוחה ובזמן מאמץ. אם הטמפרטורה גבוהה, יש לנקות אבק מגוף הקירור ולחדש את המשחה הטרמית על גבי המעבד.

כשל בספק הכוח (Power Supply Overload): ספק הכוח אינו מסוגל לספק את הספק הוואט (Wattage) הנדרש עבור כרטיס המסך והמעבד במצבי שיא עבודה, או שרכיביו הפנימיים התחממו יתר על המידה עקב מאוורר ספק תקוע.

פתרון: ודא שמאוורר ספק הכוח מסתובב בחופשיות. במידת הצורך, החלף את ספק הכוח לספק איכותי בעל הספק מתאים (למשל ספק בתקן 80+ Gold עם הספק וואט המתאים לכרטיס המסך).

🖨️ קטגוריה 3: ציוד היקפי ומדפסות (Printers & Peripherals)

3.1 תקלה: המדפסת מסרבת להדפיס ומסמכים תקועים בתור (Spooler Crash)

סימפטומים: ניסיון הדפסה מחזיר שגיאה, מסמכים תקועים במצב "Deleting..." או "Spooling" בתור ההדפסה ואינם נמחקים, והמדפסת מציגה סטטוס שגוי.

אבחון ופתרון מקיף - פרוטוקול "איפוס תור הדפסה אגרסיבי":
כאשר שירות ה-Print Spooler מסרב למחוק מסמך פגום או תקוע, יש לבצע ניקוי ברמת המערכת דרך ה-CMD כמנהל:

:: 1. עצירת שירות המדפסות הראשי של Windows
net stop spooler

:: 2. מחיקה כפויה של כל קבצי ההדפסה הזמניים (קבצי SPL ו-SHD) מתיקיית המערכת
del /Q /F /S "%systemroot%\System32\Spool\Printers\*.*"

:: 3. הפעלת שירות המדפסות מחדש
net start spooler


לאחר ביצוע פקודות אלו, תור ההדפסה יתנקה לחלוטין. ניתן לשלוח מחדש את המסמך להדפסה.

3.2 תקלה: מדפסת רשת מוגדרת כ-Offline (לא מקוונת) פתאום

סימפטומים: מדפסת המחוברת ב-Wi-Fi או בכבל רשת ארגוני מופיעה במצב אפור ולא זמינה, למרות שהיא דולקת ומחוברת פיזית לרשת.

אבחון ופתרון מקיף:

שלב א': ביטול מצב Offline וירטואלי

כנס אל הגדרות 🡪 מכשירים ומדפסות (או "מדפסות וסורקים").

לחץ קליק ימני על המדפסת שלך 🡪 בחר ב-Open print queue (פתח את תור המדפסת).

בתפריט העליון לחץ על לשונית Printer (מדפסת).

ודא שאין סימון V ליד Use Printer Offline (השתמש במדפסת במצב לא מקוון). במידה ויש - לחץ עליו להסרתו.

שלב ב': עדכון יציאת IP (הסיבה הנפוצה ביותר ברשתות DHCP)

אם הראוטר הביתי שינה את כתובת ה-IP של המדפסת, המחשב ימשיך לחפש אותה בכתובת הישנה:

הדפס "דוח הגדרות רשת" ישירות ממסך המדפסת כדי לזהות את כתובת ה-IP הנוכחית שלה (לדוגמה 192.168.1.15).

במחשב: כנס להגדרות 🡪 מדפסות וסורקים 🡪 בחר במדפסת 🡪 לחץ על Printer Properties (מאפייני מדפסת).

עבור ללשונית Ports (יציאות) ואתר את היציאה המסומנת ב-V.

אם היציאה היא מסוג WSD Port או שה-IP הרשום בה אינו תואם ל-IP הנוכחי של המדפסת:

לחץ על כפתור Add Port (הוסף יציאה).

בחר ב-Standard TCP/IP Port ולחץ על New Port.

הזן את כתובת ה-IP החדשה של המדפסת וסיים את האשף.

3.3 תקלה (נוספת): מקלדת/עכבר מגיבים בדיליי קיצוני או קופאים (USB Selective Suspend)

סימפטומים: עכבר אלחוטי או חוטי "קופץ" על המסך, מקלדת לא מקלידה מספר שניות ואז פולטת רצף תווים מהיר, או התקני USB מתנתקים באופן אקראי.

אבחון ופתרון מקיף:

מנגנון חסכון באנרגיה של Windows: המערכת מכבה את מתח ה-USB באופן סלקטיבי כדי לחסוך בחשמל.

פתרון:

לחץ Windows + R, הקלד powercfg.cpl ולחץ Enter (אפשרויות צריכת חשמל).

ליד תוכנית החשמל הפעילה שלך, לחץ על Change plan settings (שנה הגדרות תוכנית) 🡪 Change advanced power settings (שנה הגדרות צריכת חשמל מתקדמות).

גלול מטה והרחב את USB settings 🡪 USB selective suspend setting.

שנה את ההגדרה ל-Disabled (מושבת) עבור סוללה וחשמל.

3.4 תקלה (נוספת): סורק משרדי לא מזוהה (TWAIN/WIA Drivers Issue)

סימפטומים: תוכנת הסריקה מציגה שגיאה "Scanner not found" או "Cannot communicate with scanner", למרות שהסורק דולק ומחובר.

אבחון ופתרון מקיף:

סוגי דרייברים: סורקים ב-Windows משתמשים בשני מנגנונים: TWAIN (דרייבר יצרן מתקדם) או WIA (מנגנון הסריקה המובנה של מיקרוסופט).

בדיקת שירות WIA במערכת:

לחץ Windows + R, הקלד services.msc ולחץ Enter.

גלול מטה ואתר את השירות בשם Windows Image Acquisition (WIA).

ודא שסוג האתחול מוגדר כ-Automatic והסטטוס הוא Running.

במידה והוא דולק, בצע קליק ימני ולחץ Restart כדי לרענן את קישוריות הסורקים.

📧 קטגוריה 4: תוכנות, אופיס, ו-Outlook

4.1 תקלה: Outlook קורס בפתיחה, תקוע על "Processing", או לא מחפש מיילים

סימפטומים: תוכנת האאוטלוק אינה עולה, נתקעת בשלב הטעינה הראשונית, או שתיבת החיפוש אינה מחזירה תוצאות כלל.

אבחון ופתרון מקיף:

שלב א': בדיקת תוספים פגומים (Safe Mode)

תוספי צד שלישי (כמו אנטי-וירוס, Zoom, Webex) הם הגורם העיקרי לקריסות אאוטלוק:

לחץ על מקשי Windows + R, הקלד: outlook /safe ולחץ Enter.

אם התוכנה נפתחת בצורה תקינה במצב זה:

גש אל קובץ (File) 🡪 אפשרויות (Options) 🡪 תוספות (Add-ins).

בתחתית החלון, תחת תפריט Manage, בחר ב-COM Add-ins ולחץ על Go.

בטל את הסימון מתוספי צד שלישי שאינם חיוניים ולחץ אישור.

שלב ב': תיקון קובץ הנתונים הפגום (OST/PST Rebuilding)

אם האאוטלוק חווה ניתוקים או קריסות, קובץ הנתונים המקומי של המיילים ככל הנראה נפגם:

סגור את תוכנת Outlook לחלוטין.

לחץ Windows + R, הדבק את הנתיב הבא ולחץ Enter:

%localappdata%\Microsoft\Outlook


אתר את הקובץ המזוהה עם כתובת המייל שלך ומסתיים בסיומת .ost.

שנה את שם הקובץ ל-filename.ost.old (שינוי שם מונע אובדן מידע ומאלץ את המערכת לבנות קובץ חדש).

פתח מחדש את Outlook. התוכנה תמשוך מחדש בצורה נקייה ויציבה את כל המיילים משרת הדואר (Exchange/Office365).

שלב ג': בנייה מחדש של אינדקס החיפוש (Search Index Rebuild)

אם חיפוש המיילים פסק מלעבוד:

בתוך Outlook, כנס אל File 🡪 Options 🡪 Search.

לחץ על כפתור Indexing Options 🡪 לחץ על כפתור Advanced (מתקדם).

תחת לשונית Index Settings, לחץ על כפתור Rebuild (בנה מחדש).

שים לב: תהליך זה מתבצע ברקע ועשוי לקחת מספר שעות (בהתאם לנפח המיילים), במהלכן החיפוש לא יעבוד, אך בסיומו הבעיה תיפתר לחלוטין.

4.2 תקלה: Microsoft Teams / תוכנות אופיס מתנהגות מוזר או קופאות

סימפטומים: אפליקציית Teams נתקעת על מסך לבן, מציגה סטטוס לא מעודכן או אינה נטענת. תוכנות Word ו-Excel קורסות בפתאומיות.

אבחון ופתרון מקיף:

שלב א': ניקוי מטמון (Cache) אגרסיבי לאפליקציית Teams

סגור את Teams לחלוטין (קליק ימני על סמל האפליקציה בשורת המשימות למטה 🡪 בחר Quit).

לחץ על מקשי Windows + R, הדבק את הנתיב הבא ולחץ Enter:

%appdata%\Microsoft\Teams


מחק את כל הקבצים והתיקיות הנמצאים בתוך נתיב זה (אין חשש, מדובר בקבצי מטמון זמניים בלבד. היסטוריית הצ'אטים שמורה בענן).

פתח מחדש את Teams. האפליקציה תבצע סנכרון נקי ומהיר מול השרת.

שלב ב': תיקון מקוון מקיף לחבילת ה-Office

לחץ Windows + R, הקלד appwiz.cpl (תוכניות ותכונות) ולחץ Enter.

מצא ברשימה את חבילת ה-Microsoft 365 (או Microsoft Office) שלך.

בחר בה ולחץ על כפתור Change (שנה) בחלק העליון.

בחלון שיפתח, בחר באפשרות Online Repair (תיקון מקוון) ולחץ על Repair. אפשרות זו מורידה מחדש את קבצי הליבה הפגומים ופותרת בעיות קריסה קשות.

4.3 תקלה (נוספת): קובץ Word/Excel נעול לעריכה על ידי משתמש אחר

סימפטומים: ניסיון לפתוח קובץ משותף ברשת מציג הודעה: "File is locked for editing by another user", למרות שאף משתמש אינו פותח אותו באותו רגע.

אבחון ופתרון מקיף:

קבצי בעלות נסתרים (Owner Files): כאשר קובץ אופיס נפתח, Windows מייצר קובץ זמני מוסתר באותה התיקייה ששמו מתחיל ב-~$ (לדוגמה ~$Document.docx). אם האופיס נסגר בצורה לא תקינה (למשל עקב קריסה או ניתוק רשת), קובץ הבעלות נשאר בתיקייה ונועל אותה.

הפתרון:

פתח את סייר הקבצים 🡪 לשונית View (תצוגה) 🡪 סמן ב-V את Hidden items (פריטים מוסתרים).

כנס לתיקייה שבה נמצא הקובץ הנעול.

אתר את הקובץ הזמני המתחיל ב-~$ ומחק אותו פיזית מהמערכת. הקובץ המקורי ישתחרר מיד לנעילה.

4.4 תקלה (נוספת): שגיאות הפעלת רישיון אופיס/ווינדוס (KMS/MAK Activation)

סימפטומים: הודעות במערכת "This copy of Windows is not genuine" או "Product Activation Required" בתוכנות האופיס, למרות שיש רישיון ארגוני תקין.

אבחון ופתרון מקיף:
בסביבות ארגוניות, מחשבים נרשמים מול שרת רישוי פנימי (KMS). במידה והרישום נכשל, ניתן לאלץ רישום מחדש דרך ה-CMD כמנהל:

:: אילוץ רישום מחדש של מערכת ההפעלה Windows
slmgr.vbs /ato

:: בדיקת סטטוס הרישיון המפורט של ווינדוס
slmgr.vbs /dlv


במידה ומדובר ברישיון אופיס, נווט לתיקיית ההתקנה של האופיס (לדוגמה C:\Program Files\Microsoft Office\Office16) והרץ:

cscript ospp.vbs /act


🔒 קטגוריה 5: אבטחת מידע, הרשאות ואחזור מידע

5.1 תקלה: ננעלתי מחוץ למחשב (שכחתי סיסמה / בעיית PIN)

סימפטומים: שגיאות כניסה כגון "Your PIN is no longer available" או "The username or password is incorrect" מונעות את המעבר מעבר למסך הנעילה.

אבחון ופתרון מקיף:

שלב א': כניסה באמצעות סיסמה חלופית (משתמש מקומי / Microsoft Account)

במסך הכניסה של Windows, מתחת לשדה ה-PIN, לחץ על קישור Sign-in options (אפשרויות כניסה) ובחר בסמל המפתח (כניסה באמצעות הסיסמה המלאה של החשבון) במקום ה-PIN.

שלב ב': איפוס פיזי של מנגנון ה-PIN (Windows Hello Issue)

אם שירות הקריפטוגרפיה של Windows נפגם ומונע שימוש ב-PIN, יש למחוק את תיקיית האבטחה המקומית המאחסנת אותו:

היכנס למחשב באמצעות הסיסמה הרגילה (או במצב בטוח במידת הצורך).

פתח את סייר הקבצים ונווט לנתיב הבא (ייתכן שתידרש לאשר הרשאות מנהל בכניסה לכל תיקייה):

C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\Ngc


מחק את כל תוכן תיקיית Ngc לחלוטין.

בצע הפעלה מחדש למחשב, כנס להגדרות החשבון והגדר PIN חדש בצורה נקייה ותקינה.

שלב ג': חשבון נעול בסביבה ארגונית (Active Directory Security)

בסביבת דומיין, נעילת משתמש מתרחשת לרוב עקב ניסיונות אימות שגויים חוזרים ונשנים.

טיפ IT Tier 3: הגורם השכיח ביותר לנעילת חשבון חוזרת ונשנית של עובד הוא מכשיר ישן (כגון סמארטפון שהוחלף או מחשב נייד בתיק) שמנסה להתחבר לרשת האלחוטית של הארגון או לתיבת הדואר עם הסיסמה הישנה של המשתמש.

הפתרון: כבה את חיבור הרשת (Wi-Fi/נתונים) בכל המכשירים הניידים של המשתמש, בקש ממנהל המערכת לבצע Unlock לחשבון ב-Active Directory, והזן את הסיסמה החדשה בכל המכשירים בזה אחר זה.

5.2 תקלה: נמחקו לי קבצים בטעות / קבצים נעלמו (Shadow Copies)

סימפטומים: משתמש מחק תיקייה בטעות, ביצע דריסה ("Save Over") של מסמך חשוב, או שקובץ נעלם מהכונן המקומי.

אבחון ופתרון מקיף:

שלב א': שחזור באמצעות גרסאות קודמות (VSS - Volume Shadow Copy)

מנגנון ה-VSS של Windows יוצר נקודות שחזור שקטות לקבצים ברקע:

נווט אל התיקייה שבה ישב הקובץ המקורי (לדוגמה תיקיית "Documents").

קליק ימני על התיקייה (או על הקובץ שנדרס) 🡪 בחר באפשרות Restore previous versions (שחזר גרסאות קודמות).

בחלון שיפתח, תראה רשימת גרסאות היסטוריות לפי תאריכים ושעות.

בחר בגרסה המתאימה, לחץ על Open כדי לוודא שזהו הקובץ המבוקש, וגרור אותו בבטחה לשולחן העבודה.

שלב ב': בדיקת סנכרון ענן (OneDrive Client Sync Issues)

במערכות הפעלה מודרניות, תיקיות המשתמש הראשיות מסונכרנות לענן.

ודא שסמל ה-OneDrive בשורת המשימות (ענן כחול/לבן) פעיל ואינו מציג שגיאות סנכרון.

קליק ימני על קובץ שנראה כמו קישור (סמל ענן לידו) 🡪 בחר ב-Always keep on this device כדי להבטיח זמינות פיזית מקומית על הדיסק הקשיח.

5.3 תקלה: התראה על "וירוס" קופצת ללא הפסקה בצד המסך

סימפטומים: הודעות קופצות (Notifications) מטעם מערכת ההפעלה בצד ימין למטה, המדווחות על היות המחשב "נגוע בוירוסים" ומפנות לקישורים לרכישת אנטי-וירוס.

אבחון ופתרון מקיף:
זהו אינו וירוס פיזי במחשב! מדובר בתופעה של "חטיפת התראות דפדפן" (Browser Push Notifications Hijacking). המשתמש אישר בטעות שליחת התראות באתר אינטרנט זדוני/פרסומי.

שלב א' - ניקוי בדפדפן Google Chrome:

פתח את דפדפן Chrome 🡪 כנס אל הגדרות (Settings).

נווט אל Privacy and Security (פרטיות ואבטחה) 🡪 Site Settings (הגדרות אתר) 🡪 Notifications (התראות).

תחת רשימת האתרים המורשים לשלוח התראות (Allowed to send notifications), אתר דומיינים שאינם מוכרים לך (כגון כתובות מוזרות עם סיומות שונות).

לחץ על שלוש הנקודות שליד האתר הבעייתי ובחר ב-Remove (הסר) או Block (חסום).

שלב ב' - ניקוי בדפדפן Microsoft Edge:

פתח את Edge 🡪 כנס אל הגדרות 🡪 Cookies and site permissions (קובצי Cookie והרשאות אתרים) 🡪 Notifications.

מחק או חסום את האתרים הזדוניים ברשימת הרישוי. הבעיה תיפסק מיידית.

5.4 תקלה (נוספת): שגיאת "אין הרשאה" בכניסה לתיקייה (Access Denied / NTFS Ownership)

סימפטומים: ניסיון לפתוח תיקייה או קובץ מקומי מקפיץ חלון "You don't currently have permission to access this folder".

אבחון ופתרון מקיף:
הבעיה נגרמת לרוב כאשר מחברים דיסק קשיח ממחשב ישן, ומזהי האבטחה (SIDs) של המשתמש הקודם אינם תואמים למשתמש הנוכחי. יש לקבל בעלות פיזית (Take Ownership) על התיקייה:

קליק ימני על התיקייה הבעייתית 🡪 בחר Properties (מאפיינים).

עבור ללשונית Security (אבטחה) 🡪 לחץ על כפתור Advanced (מתקדם).

בשורת ה-Owner (בעלים) בחלק העליון, לחץ על קישור Change (שנה).

בשדה הריק, הקלד את שם המשתמש הנוכחי שלך או פשוט הקלד Administrators ולחץ על Check Names כדי לאמת, ואז OK.

סמן ב-V את האפשרות: Replace owner on subcontainers and objects (החלף בעלים עבור גורמים מכילים משניים ואובייקטים).

לחץ על Apply ואישור. ההרשאות ייבנו מחדש עבור המשתמש שלך.

5.5 תקלה (נוספת): כונן חיצוני דורש פרמוט (RAW File System)

סימפטומים: חיבור כונן USB חיצוני מקפיץ הודעה "You need to format the disk before you can use it", או שהכונן מוצג במנהל הדיסקים תחת פורמט RAW במקום NTFS/exFAT.

אבחון ופתרון מקיף:

מקור הבעיה: טבלת הקבצים של הכונן נפגמה (בדרך כלל עקב שליפה לא בטוחה של הכונן בזמן כתיבה).

דגש קריטי: לעולם אין ללחוץ על "Format", שכן הדבר ימחק את כל המידע!

הפתרון - שימוש בכלי התיקון המובנה:

פתח את ה-CMD כמנהל מערכת.

זהה את אות הכונן החיצוני שלך (לדוגמה כונן E:).

הרץ את הפקודה הבאה לתיקון טבלת המחיצות הלוגית של הדיסק:

chkdsk E: /f


ברוב המקרים, הכלי ישחזר את מערכת הקבצים המקורית ללא כל אובדן מידע.

🔌 קטגוריה 6: ניהול אנרגיה וסוללה במחשבים ניידים

6.1 תקלה: סוללת הלפטופ מתרוקנת במהירות קיצונית

סימפטומים: המחשב הנייד מאבד עשרות אחוזי סוללה תוך זמן קצר, או נכבה בפתאומיות כאשר הוא מגיע ל-20%-30% סוללה.

אבחון ופתרון מקיף:

שלב א': הפקת דוח בריאות סוללה מפורט (Battery Report)

במערכות Windows מובנה כלי ייעודי לאבחון בריאות הסוללה ברמת החומרה:

פתח את שורת הפקודה (CMD) כמנהל מערכת.

הקלד את הפקודה הבאה ולחץ Enter:

powercfg /batteryreport


הכלי ייצר קובץ דוח בפורמט HTML. כנס לנתיב שבו הקובץ נשמר (בדרך כלל C:\Windows\System32\battery-report.html) ופתח אותו באמצעות דפדפן אינטרנט.

גלול אל חלק ה-Installed Batteries והשוואה בין שני נתונים:

Design Capacity: הקיבולת המקורית של הסוללה בעת ייצורה במפעל.

Full Charge Capacity: הקיבולת המקסימלית שהסוללה מסוגלת להחזיק כיום בפועל לאחר השחיקה הכימית.

מסקנה מקצועית: אם ה-Full Charge Capacity נמוך מ-70% מה-Design Capacity, הסוללה שחוקה כימית ויש להחליפה פיזית כדי לפתור את בעיית פריקת האנרגיה המהירה.

6.2 תקלה: המחשב הנייד מתחמם בפתאומיות כשהוא סגור בתוך התיק

סימפטומים: המשתמש מוציא את הלפטופ מהתיק ומגלה שהוא רותח לחלוטין, המאווררים רועשים, והסוללה התרוקנה לחלוטין.

אבחון ופתרון מקיף:

בעיית Modern Standby (S0 Low Power Idle): מחשבים ניידים חדישים משתמשים במצב שינה המאפשר להם להישאר מחוברים לרשת לצורך עדכונים והתראות גם כשהמסך סגור. לעיתים המחשב "מתעורר" בתוך התיק עקב עדכון רקע, אינו מסוגל לפזר חום בתוך התיק הסגור, ומגיע לחום קיצוני.

הפתרון המקצועי: הגדרת המחשב למעבר למצב Hibernate (שינה עמוקה - S4) במקום Sleep בעת סגירת המכסה:

לחץ Windows + R, הקלד powercfg.cpl ולחץ Enter.

בתפריט הצדדי לחץ על Choose what closing the lid does (בחר מה עושה סגירת המכסה).

שנה את ההגדרות עבור "On battery" ו-"Plugged in" ל-Hibernate (מצב שינה עמוק שומר את המצב על ה-SSD ומכבה לחלוטין את החשמל, מה שמונע התעוררות מקרית בתיק).

6.3 תקלה (נוספת): סוללה מחוברת לחשמל אך לא נטענת ("Plugged in, not charging")

סימפטומים: כבל הטעינה מחובר פיזית למחשב הנייד, סמל הסוללה מציג שהמטען מחובר, אך אחוז הטעינה נשאר תקוע והסוללה אינה נטענת.

אבחון ופתרון מקיף:

מקור הבעיה: דרייבר ניהול הסוללה של מערכת ההפעלה (ACPI) נכנס למצב שגיאה/קיפאון לוגי.

הפתרון - הסרה והתקנה מחדש של דרייבר הסוללה:

לחץ קליק ימני על התחל 🡪 בחר ב-Device Manager (מנהל ההתקנים).

הרחב את קטגוריית Batteries (סוללות).

קליק ימני על ההתקן בשם Microsoft ACPI-Compliant Control Method Battery 🡪 בחר ב-Uninstall device (הסר התקן).

הערה: אל דאגה, המחשב ימשיך לפעול על החשמל.

לחץ על תפריט Action בחלק העליון של מנהל ההתקנים 🡪 בחר ב-Scan for hardware changes כדי לאלץ את המערכת להתקין את הדרייבר מחדש בצורה נקייה.

נתק את המטען, חבר אותו מחדש, והסוללה תתחיל להיטען כראוי.
